Accuracy

In order to enable accurate comparisons within the simulation before and after changing certain varaibles the randomness within the simulation should be determined by a seed. This allows the user to be certain that any change to the outcome of the simulation is due to their change of variable rather than a different result due to random variation.
The images above show the random number generator functions that I found online. These take a seed and then use the seed to determine any random numbers that they output.
I then made these two variables with the function getRandInt in order to be able to easily produce a random variable at any stage within the simulation by simply calling getRandInt. The function then returns a random number between the max you specified and one inclusive.
The initial values used to calculate the profitability of the farm within the simulation aren't particularly important. In order to make the simulation of the program as accurate as possible, human set variables are always going to be too inacurate. Therefore, an optimisation program can be used to optimise the variables to give the most accurate results. This is done by having a set of variables recorded in a real life situation and the outcome of these variables is recorded.
An optimisation program can then take the initial values and constantly tweak the variables within the program until it can acheive the same results as were produced in real life. After this process the simulation should be accurate as it can reproduce events that occured within real life. The process can be repeated a few times with different events in order to ensure maximum accuracy. The github repository for one such open source optimisation program is linked in the useful links section.
